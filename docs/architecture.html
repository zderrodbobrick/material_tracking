RFID Tracking System — Architecture (UML)

Component: tracking/listener.py + tracking/storage.py. Database: database/rfid_reads.db (SQLite, WAL mode). Diagrams below model startup, normal event ingestion, edge cases, the background sweeper, and the health-check probe. Each participant acts as a swim lane (lifeline).

Participants used across diagrams
  • FX9600 — Zebra reader; pushes JSON event batches via HTTP POST.
  • Flask /tags — Flask route inside listener.py.
  • DwellTracker — orchestrator in storage.py; holds self._lock.
  • _open — in-memory dict[epc → _Session] of active sessions.
  • SQLite — persistent store: tag_reads + component_sessions.
  • Sweeper — daemon thread running _sweep_loop every 1s.
  • HealthClient — anything calling GET /healthz (e.g. python listener.py --health).

2. Normal Tag Pass — Antenna 1 → Antenna 2

Granular flow of a complete pass. Shows raw read insertion, throttling, RSSI filter, session create on first ant1, last_enter updates, ant2 edge-case rejection if earlier than entry, dwell calculation, and the POST response. 

sequenceDiagram
    autonumber
    participant R   as FX9600
    participant FL  as Flask /tags
    participant DT  as DwellTracker
    participant OP  as _open dict
    participant DB  as SQLite

    R  ->>  FL: POST /tags  (batch of N events)
    FL ->>  FL: parse JSON, log headers/body
    FL ->>  DT: ingest_batch(events)

    DT ->>  DT: sort events by reader timestamp
    DT ->>  DT: acquire self._lock

    loop per event
        DT ->>  DT: extract epc, antenna, peakRssi
        alt epc missing OR antenna == 0
            DT -->> DT: skip
        else RSSI fails _valid_rssi
            DT -->> DT: summary.raw_rejected++
        else
            DT ->>  DT: _last_raw[(epc,ant)] check
            alt within RAW_THROTTLE_SEC (1s)
                DT -->> DT: summary.raw_throttled++
            else
                DT ->>  DB: INSERT INTO tag_reads (...)
                DT ->>  DT: summary.raw_inserted++
            end

            alt antenna == ENTRY_ANTENNA (1)
                DT ->>  OP: lookup self._open[epc]
                alt previous pass already saw ant2
                    DT ->>  DT: _finalize(prev, COMPLETE)
                    DT ->>  DB: UPDATE component_sessions ...
                    DT ->>  OP: del self._open[epc]
                end
                alt no open session
                    DT ->>  DB: INSERT component_sessions
(first_enter_at_ant1, ...)
                    DB -->> DT: lastrowid
                    DT ->>  OP: self._open[epc] = _Session(...)
                else session exists
                    DT ->>  DB: UPDATE last_enter_at_ant1, last_enter_rssi_ant1
                    DT ->>  OP: sess.last_ant1_ts = reader_dt
                end

            else antenna == EXIT_ANTENNA (2)
                DT ->>  OP: lookup self._open[epc]
                alt no open session  (EDGE CASE)
                    DT ->>  DB: INSERT EXIT_ONLY row
(first_exit_at_ant2, ...)
                    DT ->>  OP: self._open[epc] = _Session(no entry)
                else reader_dt < first_ant1_ts
                    DT -->> DT: drop (out-of-order ant2)
                else
                    alt first ant2 read
                        DT ->>  OP: sess.first_ant2_ts = reader_dt
                    end
                    DT ->>  OP: sess.last_ant2_ts = reader_dt
                    alt session has first_ant1_ts
                        DT ->>  DT: dwell = last_ant2 - first_ant1
                        DT ->>  DB: UPDATE last_exit_at_ant2,
last_exit_rssi_ant2,
dwell_seconds
                    else  (EXIT_ONLY)
                        DT ->>  DB: UPDATE last_exit_at_ant2 only
                    end
                end
            end
        end
    end

    DT ->>  DT: release self._lock
    DT -->> FL: summary {raw_inserted, throttled, rejected, opened, closed}
    FL -->> R: HTTP 200 OK

3. Edge Case: EXIT_ONLY Session

Antenna 2 fires for an EPC with no open session. We record the event so no read is lost and rely on the sweeper to finalize. 

sequenceDiagram
    autonumber
    participant R  as FX9600
    participant FL as Flask /tags
    participant DT as DwellTracker
    participant OP as _open dict
    participant DB as SQLite
    participant SW as Sweeper

    R  ->> FL: event { antenna:2, epc:X }
    FL ->> DT: ingest_batch([event])
    DT ->> OP: self._open.get(X)  →  None
    DT ->> DB: INSERT component_sessions
status=EXIT_ONLY,
first_exit_at_ant2 = now
    DT ->> OP: self._open[X] = _Session(no entry)

    Note over R,DT: more ant2 reads arrive...
    R  ->> FL: event { antenna:2, epc:X }
    FL ->> DT: ingest_batch([event])
    DT ->> DB: UPDATE last_exit_at_ant2, last_exit_rssi_ant2

    Note over SW,DB: 30s of silence elapses...
    SW ->> SW: _sweep_once()
    SW ->> OP: idle_secs = now - last_seen_wall  >= IDLE_TIMEOUT
    SW ->> SW: first_ant1_ts is None → keep status=EXIT_ONLY
    SW ->> DT: _finalize(sess, status=EXIT_ONLY)
    DT ->> DB: UPDATE last_exit_at_ant2, status=EXIT_ONLY
    DT ->> OP: del self._open[X]

4. Background Sweeper

Independent daemon thread. Every SWEEP_INTERVAL_SEC (1s) it inspects all open sessions and finalizes any whose idle wall-clock time has crossed a threshold. This is the only path that closes sessions when no further POSTs arrive. 

sequenceDiagram
    autonumber
    participant SW as Sweeper Thread
    participant DT as DwellTracker
    participant OP as _open dict
    participant DB as SQLite

    loop every 1.0s
        SW ->> SW: time.sleep(1.0)
        SW ->> DT: _sweep_once()
        DT ->> DT: acquire self._lock
        loop for each sess in self._open
            DT ->> DT: idle = now - sess.last_seen_wall
            alt idle < IDLE_TIMEOUT_SEC
                DT -->> DT: continue (still active)
            else first_ant1_ts is None
                DT ->> DT: status = EXIT_ONLY
                DT ->> DB: UPDATE last_exit_at_ant2, status
                DT ->> OP: del self._open[epc]
            else last_ant2_ts is not None
                DT ->> DT: status = COMPLETE,
dwell = last_ant2 - first_ant1
                DT ->> DB: UPDATE last_exit_at_ant2,
dwell_seconds, status
                DT ->> OP: del self._open[epc]
            else idle >= ABANDON_TIMEOUT_SEC
                DT ->> DT: status = ABANDONED
                DT ->> DB: UPDATE status only
                DT ->> OP: del self._open[epc]
            end
        end
        DT ->> DT: release self._lock
    end

5. Health Check Probe

CLI client (python listener.py --health) opens a short HTTP request against the running listener. 

sequenceDiagram
    autonumber
    participant HC as HealthClient
    participant FL as Flask /healthz
    participant DT as DwellTracker
    participant OP as _open dict

    HC ->> FL: GET /healthz
    FL ->> DT: tracker.open_session_count()
    DT ->> OP: with lock: len(self._open)
    OP -->> DT: N
    DT -->> FL: N
    FL ->> FL: build JSON {status, uptime, open_sessions,
events_total, batches_total,
last_event_seconds_ago, db_writable}
    alt db_ok
        FL -->> HC: 200 {status: ok, ...}
    else db_ok == false
        FL -->> HC: 503 {status: degraded, ...}
    end
    HC ->> HC: print summary, exit code 0/1/2

7. State Machine — component_sessions.status

Valid transitions for a single row. Every row is born IN_PROGRESS (from ant1) or EXIT_ONLY (from ant2 with no open session).

    IN_PROGRESS --> IN_PROGRESS : ant1 read updates
last_enter_at_ant1
    IN_PROGRESS --> IN_PROGRESS : ant2 read updates
last_exit_at_ant2,
dwell_seconds
    IN_PROGRESS --> COMPLETE    : sweeper closes
(idle >= IDLE_TIMEOUT,
last_ant2 set)
    IN_PROGRESS --> ABANDONED   : sweeper closes
(idle >= ABANDON_TIMEOUT,
no ant2 ever)

    EXIT_ONLY --> EXIT_ONLY     : more ant2 reads

(status unchanged)
