RFID Tracking System — Architecture

High-level view of how a tag is tracked from the moment it passes the entry antenna to when its session is closed. Each vertical lane in the flow diagrams represents one part of the system.

1. Startup

What happens when the listener is launched.

sequenceDiagram
    autonumber
    participant U  as Operator
    participant S  as Listener
    participant DB as Database
    participant SW as Cleanup Worker

    U  ->>  S:  Start the listener
    S  ->>  DB: Open and prepare database
    S  ->>  DB: Reload any unfinished sessions
    DB -->> S:  Open sessions restored
    S  ->>  SW: Start the cleanup worker
    Note over S,SW: Listener is ready to receive tag events.

2. Normal Tag Pass

A tag enters the zone at antenna 1, travels through, and leaves at antenna 2. The dwell time is the difference between the first read at antenna 1 and the last read at antenna 2. 

sequenceDiagram
    autonumber
    participant R  as Reader
    participant S  as Listener
    participant M  as Active Sessions
    participant DB as Database

    R ->> S: Tag seen at antenna 1
    S ->> M: Open new session for this tag
    S ->> DB: Save first-enter timestamp

    R ->> S: Tag seen again at antenna 1
    S ->> M: Update last-enter timestamp
    S ->> DB: Save last-enter timestamp

    R ->> S: Tag seen at antenna 2
    S ->> M: Mark first-exit timestamp
    S ->> DB: Save first-exit timestamp

    R ->> S: Tag seen again at antenna 2
    S ->> M: Update last-exit timestamp
    S ->> DB: Save last-exit and dwell time

    Note over R,DB: Tag goes silent.
    S ->> DB: Mark session COMPLETE

3. Exit-Only Edge Case

A tag is seen at the exit antenna without ever appearing at the entry antenna. The system still records the read so nothing is lost. 

sequenceDiagram
    autonumber
    participant R  as Reader
    participant S  as Listener
    participant DB as Database
    participant SW as Cleanup Worker

    R ->> S: Tag seen at antenna 2 only
    S ->> DB: Create session marked EXIT-ONLY
    R ->> S: More antenna 2 reads
    S ->> DB: Update last-exit timestamp
    Note over R,SW: Tag goes silent.
    SW ->> DB: Close session, keep status EXIT-ONLY

4. Cleanup Worker

Runs continuously in the background. Closes sessions whose tags have gone silent so nothing stays open forever. 

sequenceDiagram
    autonumber
    participant SW as Cleanup Worker
    participant M  as Active Sessions
    participant DB as Database

    loop Every second
        SW ->> M: Check each open session
        alt Tag still active
            SW -->> SW: Leave session open
        else Tag was seen at both antennas, then went silent
            SW ->> DB: Close as COMPLETE with dwell time
        else Tag only seen at exit antenna
            SW ->> DB: Close as EXIT-ONLY
        else Tag only seen at entry antenna and disappeared
            SW ->> DB: Close as ABANDONED
        end
    end

5. Health Check

An operator or monitor asks the listener if it is alive and well.

sequenceDiagram
    autonumber
    participant U  as Operator
    participant S  as Listener

    U ->> S: Ask for health status
    S ->> S: Check uptime, open sessions, last event, database
    alt Healthy
        S -->> U: Status OK
    else Database problem
        S -->> U: Status DEGRADED
    end

7. Session Lifecycle

The possible states a session can be in.

stateDiagram-v2
    [*] --> InProgress  : First seen at entry antenna
    [*] --> ExitOnly    : First seen at exit antenna only

    InProgress --> InProgress : Tag still being seen
    InProgress --> Complete   : Saw both antennas, then went silent
    InProgress --> Abandoned  : Never reached exit, then went silent

    ExitOnly --> ExitOnly : More exit reads
    ExitOnly --> ExitOnly : Closed by cleanup worker

    Complete  --> [*]
    Abandoned --> [*]
    ExitOnly  --> [*]
